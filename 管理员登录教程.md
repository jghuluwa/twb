# Therabo 网站 —— 管理员后台登录教程

> 本文档说明**管理员登录页面在哪里**、**用什么账号登录**，以及部署时需要注意的几件事。
> 你可以直接把这份文档交给部署的 agent。

---

## 一、后台登录地址在哪里？

管理后台**没有单独的网址**，它和主站是同一个网站，只是在网址后面加一个 `#admin`。

打开浏览器，在你的网站地址后面加上 `#admin` 即可：

```
https://你的域名/#admin
```

**举例：**

| 你的网站地址 | 管理后台登录地址 |
| --- | --- |
| `https://therabo.com` | `https://therabo.com/#admin` |
| `https://www.tongweibao.cn` | `https://www.tongweibao.cn/#admin` |
| `http://123.45.67.89` （IP 部署） | `http://123.45.67.89/#admin` |

> ⚠️ 注意是 `#admin`（井号，hash 路由），**不是** `/admin`。
> 打开后会看到一个标题为「**Therabo 管理后台 / Admin Console**」的登录框。
>
> 登录后想回到前台主站，点登录框下方的「返回前台主站」即可。

## 二、用什么账号密码登录？

管理员账号是**部署时自动创建的**，来自 `.env` 配置文件里的这两项：

```bash
ADMIN_BOOTSTRAP_USERNAME=admin          # 用户名，默认就是 admin
ADMIN_BOOTSTRAP_PASSWORD=你设置的密码    # 密码，部署时自己填
```

- **用户名**：默认是 `admin`（除非你在 `.env` 里改了 `ADMIN_BOOTSTRAP_USERNAME`）。
- **密码**：就是部署时 `.env` 里 `ADMIN_BOOTSTRAP_PASSWORD` 填的那个值。

如果你**没有改过** `.env`，那密码就是文件里预设的占位值（例如 `please-change-on-first-login` 或 `changeme-now`）—— 这种情况**必须**马上登录改掉，否则不安全。

## 三、如果忘了/不知道当时设的密码，给 agent 的操作指引

把下面这段直接交给部署的 agent 执行即可。

### 情况 A：还没正式部署，或可以重新初始化数据库

1. 在 `.env` 文件里设置一个你自己的强密码：

   ```bash
   ADMIN_BOOTSTRAP_USERNAME=admin
   ADMIN_BOOTSTRAP_PASSWORD=一个你自己的强密码
   ```

2. 管理员账号是在首次启动时由 `node dist/seed.js`（seed 脚本）创建的。
   该脚本**幂等**：如果数据库里**已经有**管理员账号，它会跳过、**不会**用新密码覆盖。
   - 所以如果数据库是全新的（还没创建过 admin），改完 `.env` 重启服务即可生效。
   - 如果 admin 已经存在但你想用新密码重建，需要先删除旧账号（见情况 B）。

### 情况 B：数据库里已有 admin，但密码丢了（不想清库）

数据存在 SQLite 数据库里（容器内路径通常是 `/app/data/therabo.db`）。让 agent 执行：

1. 生成新密码的 bcrypt 哈希（在项目 `server/` 目录下）：

   ```bash
   node -e "import('./dist/lib/auth.js').then(m => m.hashPassword('新密码').then(h => console.log(h)))"
   ```

   会打印出一串以 `$2` 开头的哈希字符串。

2. 把这个哈希写进数据库，覆盖 admin 的密码：

   ```bash
   sqlite3 /app/data/therabo.db "UPDATE admin_users SET password_hash='上一步生成的哈希' WHERE username='admin';"
   ```

3. 用「`admin` + 新密码」登录 `https://你的域名/#admin`。

> 更详细的部署/数据库说明见仓库里的 [DEPLOY.md](DEPLOY.md)。

## 四、登录之后要做的第一件事

**立刻修改默认密码 / 重建管理员账号**，不要继续用 `.env` 里的初始密码。

登录后台后，进入左侧菜单的「**管理员账号 / Users**」页面：

- 新建一个属于你自己的管理员账号（角色选 `admin`），
- 然后删除初始的 `admin` 账号（或修改其密码）。

> 角色说明：
> - `admin`：拥有全部权限（含管理其它管理员、改密码）。
> - `editor`：可管理产品、查看订单，但**不能**改管理员账号。

## 五、快速排查：打开 `#admin` 没反应？

| 现象 | 可能原因 / 处理 |
| --- | --- |
| 加了 `#admin` 还是普通主站页面 | 确认网址里是 `#admin`（井号），不是 `/admin`；刷新一次页面 |
| 登录框出现，但提示「用户名或密码错误」 | 用户名/密码不对 —— 核对 `.env` 里的 `ADMIN_BOOTSTRAP_*`，或按第三节重置 |
| 登录时报网络/接口错误 | 后端 API 服务（server）没起来，或前端没连到后端；让 agent 检查容器是否都在运行 |
| 整个网站打不开 | 先解决主站部署问题，参考 [DEPLOY.md](DEPLOY.md) |

---

### 一句话总结

> **后台地址 = 你的网址 + `#admin`；账号 = `admin` + 部署时 `.env` 里设的 `ADMIN_BOOTSTRAP_PASSWORD`；登录后第一件事是改密码。**
